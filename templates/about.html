Kiosk Checks is a log parser for Kiosk Logon or Provisioning issues. It can analyze the logs collected with the help of TSSv2 [.\TSSv2.ps1 -Start -UEX_logon]



Applicable scenarios

- Kiosk user logs on but sees the normal desktop instead of the configured applications

- Kiosk user gets an assigned access view but has the wrong applications

- Kiosk user is missing applications despite them being configured

- Kiosk provisioning package / xml cannot be applied


What it does

- Looks for the last provisioning error

- Checks if the AssignedAccess Manager Service is disabled

- Checks if UWP Application AUMIDs can be found on the system - this highlights whether they are not installed or misspelled

- Checks if UWP applications are installed/registered for the users they are assigned to

- Looks for every Kiosk configuration found in the registry and lists it in a readable format

- Extracts any Multi-App Kiosk or Shell Launcher XML found in the registry; it can then be applied on a test environment

- Converts and parses the ETL provider trace, filtering out non-assigned access events and highlighting any errors (non-zero HRESULTS)


What it cannot do

- If the kiosk configuration is a assigned to a group instead of an individual user, it cannot check the installed applications; in this case you'll have to identify the profile in the report and manually look for the apps

- Find inconsistencies in the xml (e.g. path names / wrong tile placement etc...)
