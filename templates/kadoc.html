Note: Kiosk Assistant is an INTERNAL ONLY tool. It is developed by a Support Escalation Engineer in Windows Commercial Support, and is not authorized for distribution to anyone outside of Windows Commercial Support.


KioskAssistant is a UI tool that facilitates the creation Multi-App Kiosk environments. Its purpose is to speed up the configuration of XML files as well as prevent any mistakes by employing thorough checks and automatic XML Formatting. This is ideal for setting up test environments.

To use, simply extract and make sure you have .NET Desktop Runtime 5 installed: Download .NET 5.0 (Linux, macOS, and Windows) (microsoft.com)


Usage

 

The main window is split into the Accounts View and the Application Profiles view – to generate an XML you need to have at least one of each, then associate them with each-other

 

 

An account can be a Local User/Group, AD User/Group, AAD User/Group, Global Profile or AutoLogonAccount. Each account will need to have one application profile associated with it (it can be the same application profile for one or more accounts).

 

To add an account, use the buttons on the left view. Click "Add Application Profile" to begin configuring the Kiosk profiles.

 

 

The Application Profile Window is split as follows:

 

 

- UWP Application List

- Win32 Application Panel

- Link file Panel

- Group Configuration

 

An application profile needs at least an App Group (Start Menu Group). To create a group, you need to select at least 1 Application to allow.

 

To allow an UWP Application, simply tick the box corresponding to it.

To allow a Win32 Application, click the Add button above the pannel and select an executable (.exe file)

To add an Win32 Application application to a start menu group, it needs to be associated with a Link (.lnk) file. To add a Link (.lnk) file click the Add button above its panel.

 

Note: The .lnk file needs to point back to an .exe that was already added to the Win32 panel. If no corresponding .exe can be found, the app will notify you and the link won't be added.

 

 

 

Once you have selected your applications, you can organize them in App Groups. Click the Add and Configure App Group button.

 

Multi-App Groups work the same as start menu groups – they can have a maximum of 24 Cells (6x4). The App Group Window will contain a preview that shows you the placement of each tile you add according to its size and column/row:

 

 

 

 

Trying to place a tile at an invalid position is not possible and you will be notified to adjust.

 

After you've finished the group, click "Finish" to go back to the App Profile screen. You will now see the group in the list. Leftover applications will remain allowed and they can be added to a different group. Applications that are not added to any group will still be allowed but will not be shown on the start menu.

 

 

Once you have your groups settled you can configure additional settings via the tickboxes in the upper middle or an auto-launch app with arguments [optional].

 

The Profile can then be saved with an automatically generated GUID (visible in the upper left) and will be displayed on the main screen.
